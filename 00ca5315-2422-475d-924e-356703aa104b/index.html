0023 今日待办和今日完成-多插件

  2022-10-18     Published·Blog·Note/Obsidian·Note/Note·Note/Article

每天的日报里显示两个栏目，一个今日待办，一个今日完成。

单纯查询待办事项和已完成事项都比较简单。举例（使用时请标记对应代码块的语言类型）：

Query 待办

Query 完成

Dataview 待办

Dataview 完成

任务相关的事情加一个 Tasks 插件会很方便，它也提供查询，而且语法很友好

Tasks 待办

Tasks 完成

但是这样并没有解决问题——今天。当然待办（未完成）这里可以宽泛一些，毕竟完成以后就到今日完成里去了。

那么今日完成呢？我们得把日期放入查询，就是必须这一天完成的才算。而且日期不能是动态的，得写死，不然每天看都是今天的日期……这符合我的使用习惯，但是每天新建一个日报就是另一回事儿了。日期应该写死为文档创建时的日期。

对了，用模板，在创建的时候生成日期写进去。所以用 Templater。（其实用 QuickAdd 也可以，不过大同小异，我只讲一个）。

下一个问题，程序怎么知道这个任务是哪天完成的啊？程序不知道，你得在完成任务的时候标记上是哪一天完成的。是不是很麻烦？没事儿，Tasks 插件中一个选项的事情，而且 Tasks 的标记方式和 Dataview 认可的标记方式完全相同，所以……很赞！

思路有了，开始写模板：

Query 今日完成

Dataview 今日完成

这里有个小坑，Dataview 中的时间相关属性是时间对象，不是字符串

Tasks 今日完成
