Templater 是个很强大的插件$1

一说到模板，第一印象就是从模板新建文件，而其实很多时候模板更像是插入片段（Snippet）$1

一个很简单的需求，我们可能需要插入时间日期，于是新建一个模板$1

<% tp.date.now() %>
``$1

然后在需要的时候调用就可以实现插入时间了$1

我们也可以格式化时间，可以参照这里：[https://momentjs.com/docs/#/displaying/](https://momentjs.com/docs/#/displaying/$1

反正 Temlater 中时间日期部分用的 [Moment.js](https://momentjs.com/docs/) ，多数东西都可以照搬，能弄出一些有趣的玩法$1

### 动态显$1

我的入口笔记是不变的（不是每天新建），而上面方法插入的是静态内容，就是插入以后就不变化了。显然这不适合我，我要笔记永远显示当天的日期，也很简单$1

```eta
<%+ tp.date.now() %>
``$1

注意区别，就是前面多了一个加号。这个代码可以直接写在笔记中。这被称作为动态模板，在每次预览笔记的时候渲染一次。所以每次预览，这里都会显示为当前日期$1

注意一个文档中没有提及的点：**动态模板输出的内容会当作文本显示，各种标记，换行均会显示为纯文本**。我被这个坑折磨了好一会儿的$1

### 显示农$1

这要用到 JS。核心代码很简单$1

```js
new Date().toLocaleDateString('zh-CN-u-ca-chinese')
``$1

获取日期，转换为本地日期格式，然后地区设置为中国本土日历。但是结果显示的有点怪：2022年九月20$1

一般来说我们不关注年份，所以可以把年份去掉，后面的 20 改成中文数字。意义都一样，就是看着舒服点$1

```eta
<%*
  /**
   * 上面星号表示运行代码中的 JS
   * tR 是要输出的内容，往里面追加即可
   * 下面将当前日期加入了输出，并在后面加上空格
   */
  tR += tp.date.now()+' '
  /* JS 中获取当前时间对象备用 */
  const today = new Date()
  /* 创建中文数字数组，这里我们不需要“零”，换成“日”用来显示星期几 */
  const num = ['日', '一', '二', '三', '四', '五', '六', '七', '八', '九']
  /**
   * 第一行：获取农历日期
   * 第二行：替换掉年份
   * 第三行：将日期替换为中文写法
   * 末尾：加上空格分隔
   */
  tR += today.toLocaleDateString('zh-CN-u-ca-chinese')
    .replace(/(\d+)\s*?年\s*/, '')
    .replace(/月(\d+)/, (m, s)=>{
      s = Number(s)
      if(s===20) return '月 二十'
      if(s===30) return '月 三十'
      const lastNum = num[s%10]
      if(s<10) return '月 初'+lastNum
      if(s<20) return '月 十'+lastNum
      if(s<30) return '月 廿'+lastNum
    })
    +' '
  /* 加上星期几 */
  tR += '星期'+num[today.getDay()]
%>
``$1

我添加了注释，这样即便不懂代码，也可以稍微修改一些细节。最终效果$1

2022-10-15 九月 二十 星期$1

我会把它放在标题中显示，这样醒目一些$1

这么长能不能当作动态模板？这个问题很赞，理论上可以，就是把 `*` 换成 `+`，但实际上不可行，因为一些特殊字符干扰了解析。这时候最好使用用户脚本，我们后面再聊。
