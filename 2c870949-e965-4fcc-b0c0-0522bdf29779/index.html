Obsidian 本身有笔记漫游功能，这个功能很好，能让我回忆起一些已经遗忘的知识$1

但，它是在所有笔记中漫游。那我的模板文件夹里的东西也被拿出来漫游就挺奇怪的。以及漫游到我每天都看的列表文件也挺奇怪的$1

Obisidan，垃圾捏！所以自己写一个！（方便捏$1

注意事$1

这个必须在给代码之前讲一下。实现很简单，找到所有的笔记，过滤一下，生成一个随机数，选中对应笔记，跳转…$1

问题就出在跳转上，这篇用来漫游的笔记打开就随机跳转了，功能上完全符合，但没法在 Obsidian 里面编辑，所以以后修改的时候要用第三方编辑器$1

小瑕疵，反正设定好以后也没啥可改的。（反正改代码我都开 VS Cod$1

代$1

新建一篇笔记，建立一个 dataviewjs 代码块，放入以下代码$1

```js
/**

  • 下面数组是要排除的文件夹
  • 需要把每一个文件夹及子文件夹逐一列出
  • 确实麻烦了点，但这样不用学习正则，也避免一些特殊字符的干扰
  • 如果你会正则，那大概你也能自行修改过滤方法
    /
    const pathBlackList = [
    ‘Templates/Obsidian’,
    ‘Templates/Scripts’,
    ‘Templates/Templater’,
    ]
    /    *
  • 获取所有页面并过滤
  • dv.pages() 的参数本身就可以过滤
  • 这里使用专门的过滤函数是为了更复杂的过滤，根据需求选择
  • filter 方法和 dataview 提供的 where 方法几乎可以互换
    /
    const pages = dv.pages().filter(e=>{
    /    *
    • 第一个条件是判断文件后缀，排除图片等文件
    • 第二个是判断路径不在上面数组中，如果去掉感叹号就成了白名单过滤了
      /
      return e.file.ext === ‘md’
      && !pathBlackList.includes(e.file.folder)
      })
      /       生成随机数，选取笔记，获取对应的地址 /
      const path = encodeURIComponent( pages[ Math.round(Math.random()pages.length) ].file.path )
      /
  • 打开当前 vault 中的对应笔记
  • 注释掉下面代码就不会跳转了，就可以在 Obsidian 中编辑了
    */
    open(‘obsidian://open?file=’+path)
    ``$1

说$1

过滤条件有很多设定方式，实在难以逐条举例，为了方便没有 JS 和正则基础的朋友使用，所以用了最稳妥的方案。
