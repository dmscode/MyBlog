效果展示

功能说明

  • 把符合条件的笔记按如上格式展示出来（Dataviewjs 查询）
  • 鼠标放在缩略图上会显示说明信息
  • 点击标题或者标题下面的路径直接跳转到笔记

数据对应

缩略图

对应 Front matter 中的：banner、cover 字段，如果没有这两个属性，则使用来源地址的网站缩略图。

如果上述信息都不存在，则显示一张占位图，如上图中最后一篇。

来源地址

对应 Front matter 中的：source、url、link、href 字段，用哪个名称都可以，优先级从高到低。

标题

标题使用的是笔记标题，在 Obsidian 中即此篇笔记的的文件名，一般来说大家不会让这个名称过长。

描述

对应 Front matter 中的：desc、description、summary、title 字段。这里的 title 只是一个字段，尤其是摘抄过来的文章，这是来源页面的标题，可能比较长，用作描述还算合理。

一些问题

关于网址缩略图

使用的是 Api 获取，第一次请求会有一个生成的时间，而 Obsidian 本身又有一些缓存，所以第一次使用可能看到一大片转圈圈，不用太理会，也许下一次打开 Obsidian 的时候就好了。

关于占位图

使用的是 https://fakeimg.pl/ ，可以根据自己的喜好修改。不把笔记名称直接输出在上面是因为这个 Api 显示中文只有一个字体（Note），不怎么好看，而且显示字数和效果问题太多。

关于不同主题

不同 obsidian 主题下的显示效果不尽一致，甚至有些主题下还可能错乱，这个就比较难以顾及了，我只能尽力维护在默认主题下的显示效果。好消息是在默认主题下还打破了编辑器默认显示宽度，可以更好的展示。

关于移动设备

我测试在手机上使用一切正常。没有 iPhone 和 iPad，期待反馈。

关于尺寸

也算是很中规中矩的设定，在电脑和移动设备上都算比较合适。不打算开放修改，也避免参数过于复杂对小白玩家不友好。

关于更新

代码段中都有更新时间，对比就可以得知我是否更新了代码，然后重新复制粘贴就好了。
