上一篇我们把收藏夹塞进了 Obsidian，简悦老板说能不能让剪藏过来的笔记也这么显示，我说好呀！反正我也有……几篇剪藏过来的笔记，原打算收拾完书签就收拾这些的，这就提前一下吧$1

效果展$1

![webclipper.png](https://global.cdn.blingwang.cn/2022/10/24/63561e6f87b5f.png$1

功能说$1

  • 把符合条件的笔记按如上格式展示出来（Dataviewjs 查询）
  • 鼠标放在缩略图上会显示说明信息
  • 点击标题或者标题下面的路径直接跳转到笔记
  • 显示创建和更新时间
  • 如果笔记有来源链接，则显示 🔗 图标，点击图标打开源链$1

数据对$1

缩略$1

对应 Front matter 中的：banner、cover 字段，如果没有这两个属性，则使用来源地址的网站缩略图$1

如果上述信息都不存在，则显示一张占位图，如上图中最后一篇$1

来源地$1

对应 Front matter 中的：source、url、link、href 字段，用哪个名称都可以，优先级从高到低$1

标$1

标题使用的是笔记标题，在 Obsidian 中即此篇笔记的的文件名，一般来说大家不会让这个名称过长$1

描$1

对应 Front matter 中的：desc、description、summary、title 字段。这里的 title 只是一个字段，尤其是摘抄过来的文章，这是来源页面的标题，可能比较长，用作描述还算合理$1

安装方$1

这里使用了 Dataview 插件的自定义视图，所以要安装并开启插件，然后打开插件中的：Enable JavaScript Queries 选项$1

将下面两段代码保存在同一个文件夹下（这个文件夹里只放这两个文件），注意文件名称必须是 view.js 和 view.css 然后记好这个文件夹的路径，比如我们就假定为：Dataview/custom-view/WebClipper$1

Dataview/custom-view/WebClipper/view.j$1

```js
/*

  • @Author : 稻米鼠
  • @Date : 2022-10-22 09:48:06
  • @LastEditTime : 2022-10-24 13:02:07
  • @FilePath : Dataview/custom-view/WebClipper/view.js
  • @Description :
    *$1

let pages = dv.pages()
if(input && input.pages) pages = input.pages

/**

  • 获取源链接
  • @param {*} file
    /
    const get_url = file=> file.frontmatter
    ? ( file.frontmatter.source || file.frontmatter.url || file.frontmatter.link || file.frontmatter.href)
    : ‘’
    const get_desc = file=> {
    if(file.frontmatter){
    if(file.frontmatter.desc) return file.frontmatter.desc
    if(file.frontmatter.description) return file.frontmatter.description
    if(file.frontmatter.summary) return file.frontmatter.summary
    if(file.frontmatter.tile) return file.frontmatter.tile
    }
    return file.name
    }
    /     获取缩略图链接 */
    const get_screenshot = (file, url)=> {
    if(file.frontmatter){
    if(file.frontmatter.banner) return file.frontmatter.banner
    if(file.frontmatter.cover) return file.frontmatter.cover
    }
    if(url) return https://s0.wp.com/mshots/v1/${ encodeURIComponent(url) }?w=320&h=160
    return https://fakeimg.pl/320x160/?text=Note&font=lobster
    $1

/* 整理生成书签页代码 /
let code = ‘

‘
if (!pages.length) code += ‘No notes to show.’
pages.forEach(p=>{
const url = get_url(p.file)
code += <div class="webclipper-card"> <div class="webclipper-info" style="background-image: url(${ get_screenshot(p.file, url) })"> <div class="webclipper-desc">${ get_desc(p.file) }</div> </div> <div class="webclipper-title"> <a class="webclipper-icon" href="${ url }">${ url ? '🔗' : '📓' }</a> <a class="internal-link webclipper-link" data-href="${ p.file.path }" href="${p.file.path}" title="${p.file.name}">${p.file.name}</a> </div> <div class="webclipper-url"> <a data-href="${ p.file.path }" href="${ p.file.path }" class="internal-link webclipper-link" target="_blank" rel="noopener">${ p.file.path }</a> </div> <div class="webclipper-add"> Created: ${ dv.func.dateformat(p.file.ctime, 'yyyy-MM-dd HH:mm:ss') } / Update: ${ dv.func.dateformat(p.file.mtime, 'yyyy-MM-dd HH:mm:ss') } </div> </div>
})
code += </div>
/ 创建一个元素，然后把代码塞进去 */
const root = dv.el(‘div’, ‘’, {cls: ‘webclipper-index’})
root.innerHTML = code

使用方$1

上述代码保存好之后重启一下 Obsidian，确保代码被加载。然后在任意笔记中加入一个代码块，代码语言为 dataviewjs，然后粘贴如下代码$1

```js
await dv.view(‘Dataview/custom-view/WebClipper’)
``$1

很容易看出，小括号里面引号中的一长串，是我们上面所说的那个模板路径，根据自己的实际情况替换好就行了$1

预览一下效果吧，希望你的笔记数量不是很大，毕竟这条代码是展示所有笔记的$1

显然，多数时候我们的需求不是这样的，这时候我们需要第二个参数来告知我们要显示哪些笔记。使用的是 Dataview 的查询功能，我简单做几个示范$1

查看指定标签笔$1

```js
await dv.view(‘Dataview/custom-view/WebClipper’, {pages: dv.pages(‘#Webclipper’)})
``$1

如果需要修改，就修改 #Webclipper 为你需要的标签就好了$1

查看指定路$1

```js
await dv.view(‘Dataview/custom-view/WebClipper’, {pages: dv.pages(‘“Notes/folder”‘)})
``$1

把里面的 Notes\folder 修改为要显示的路径即可。这里需要注意：双层引号，如果写成单层引号会报错，注意和上面代码做区分$1

更复杂$1

如果你不懂 JS，这部分没必要看，上面两种方式可以应付大部分需求了$1

如果对 JS 有点了解，很容易看出上面就是调用函数，传递参数。第二个参数中的 pages 需要一个数组。dv.pages 查询获得的是一个笔记数组，如果用 dv.pages() 不加任何参数则获取的是所有笔记$1

数组，有一个 filter 方法，可以很方便的对数据进行过滤。所以参考一下 dataview 的文档，就能够很自由以各种条件对笔记进行筛选，比如$1

```js
await dv.view(‘Dataview/custom-view/WebClipper’, {pages: dv.pages().filter(p=>/@annote.md$/.test(p.file.path))})
``$1

这样就是显示所有文件名末尾是 @annote.md 的笔记了$1

```js
await dv.view(‘Dataview/custom-view/WebClipper’, {pages: dv.pages().filter(p=>p.file.frontmatter && p.file.frontmatter.source)})
``$1

这样就是所有具有文章来源（source）的笔记$1

一些问$1

关于网址缩略$1

使用的是 Api 获取，第一次请求会有一个生成的时间，而 Obsidian 本身又有一些缓存，所以第一次使用可能看到一大片转圈圈，不用太理会，也许下一次打开 Obsidian 的时候就好了$1

关于占位$1

使用的是 https://fakeimg.pl/ ，可以根据自己的喜好修改。不把笔记名称直接输出在上面是因为这个 Api 显示中文只有一个字体（Note），不怎么好看，而且显示字数和效果问题太多$1

关于不同主$1

不同 obsidian 主题下的显示效果不尽一致，甚至有些主题下还可能错乱，这个就比较难以顾及了，我只能尽力维护在默认主题下的显示效果。好消息是在默认主题下还打破了编辑器默认显示宽度，可以更好的展示$1

关于移动设$1

我测试在手机上使用一切正常。没有 iPhone 和 iPad，期待反馈$1

关于尺$1

也算是很中规中矩的设定，在电脑和移动设备上都算比较合适。不打算开放修改，也避免参数过于复杂对小白玩家不友好$1

关于更$1

代码段中都有更新时间，对比就可以得知我是否更新了代码，然后重新复制粘贴就好了。
