起始页的其他信息也都是自己写 JS 实现的，这些留到后面去分享。我们先看看待办列表的汇总$1

其实我有一个 Working List 的文件，里面记录我的工作任务，这是一个树状列表，按项目，然后项目中的功能点，然后具体的子任务。这样结构清晰，也非常好读。不过这是记录，而在每天开始，我想看到的就是所有下一步想去做的小任务（标记 #Next 的任务）$1

这时候就可以书写一个查询（query），前面说过怎么用了$1

task-todo: “#Next”
path:”Inbox/List/Working List”
``$1

至于为什么出问题，我也不了解。这种原生查询太弱了，而且样子也不好看，所以我既然装了 Dataview，当然是用 Dataview 了$1

```JS
task From “Inbox/List/Working List”
WHERE !completed AND econtains(tags, “#Next”)
``$1

不解说了，只要大概认识里面单词就能明白意思，并且也能自己修改吧。放在标记为 dataview 的代码块里就能使用$1

上面的语法是 Dataview Query，所以和原生的 query 语法还是有那么一些类似的。我是比较懒的学习新语言了，毕竟这东西也只是笔记中使用，那我很快就会忘掉的。既然 Dataview 支持使用 JS，我更喜欢用 JS 去写$1

正好后面关于购物的部分，我所有购物内容都放在一个文件里，然后按标题区分列表，其中有一个标题“Waitinng”，表示正在寄送中的快递。你懂的，我现在要搜索的是某个文件中某个标题下面的内容$1

道理上这里要找 section（章节），然后判断它的 subpath，但是我用上面的 query 语法写怎么也不成功。于是就自己用 JS 解决了（放在标记为 dataviewjs 的代码块里）$1

```js
const tasks = dv.page(“Inbox/List/Shopping List”) /* 指定文件 */
.file.tasks /* 文件中的所有任务 */
.filter( /* 对任务进行过滤 */
e=>
!e.completed /* 任务状态不是已完成 */
&& e.section.subpath===’Waiting’ /* 并且 章节路径是 Waiting */
)
dv.taskList( /* 按任务列表格式输出 */
dv.array(tasks), /* 把数组转换为 dataview 数组 */
false /* 不按文件分组 */
)
``$1

反正想套用应该能看出来改什么地方。
