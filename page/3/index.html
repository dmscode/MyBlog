0027 笔记中的图片管理

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

本地图片

首先，应该设置一下附件位置和存放规则——“设置——文件与链接”中有两个附件相关的选项。

推荐使用：

  • 指定的附件文件夹：就是设定一个文件夹专门用来保存附件（这里说的附件不单指图片）。这种方法附件可以集中管理。

  • 当前文件所在文件夹下指定的子文件夹中：这句话说的比较绕，就是在你某篇笔记的身边建立一个文件夹，用来存放这篇笔记相关的附件。这种方法附件就在笔记身边，方便迁移。

设定好之后，本地的图片就直接粘贴或者拖拽过来就好，文件会自动复制到附件文件夹中。（其他类型附件的处理方法相同）

更灵活的附件文件夹

如果你觉得第二种方式（当前文件所在文件夹下指定的子文件夹中）适合你，可能你要了解一下细节。Obsidian 对于附件文件夹的自动建立不支持使用变量，也就是在某一个文件夹下只能有一个附件文件夹（因为同名），这并不灵活。这时候可以借助 Custom Attachment location 插件来设定附件位置，而且支持自动文件重命名，文件名修改同步修改附件文件夹和附件名称。

图片重命名

Paste image rename 插件可以对粘贴的图片进行重命名，设定好规则可以无感自动运行，也可以每次粘入图片时提示确认。但拖拽进来的图片似乎不能触发它工作。

不过没什么，通过命令调用它可以批量处理当前笔记中所有图片。（Ctrl+P 打开命令面板，输入 Paste 找你觉得最合适的选项……

清理图片

在笔记中删除图片，并不会同步删除附件。这也不用在意，时不时的用插件清理一下就行了（两个插件任选，可能还有别的同功能插件，我就是举例）：

  • Clear Unused Images

  • File Cleaner

网络图片

网络图片直接用原始网址的话，粘贴就行了。如果希望粘贴的同时将图片保存到本地，安装 Local Images 插件并做好设置即可。也可以通过命令使用该插件批量下载所有笔记中引用的网络图片。

但是这个插件也不支持附件文件夹路径中使用变量。

使用图床

一般我是不建议的，要么很折腾，要么不是很有保障，如果图片不是很多，还是把图片放本地（和笔记放一起）吧。

如果是能保障不随机跑路的图床，或者你的图片并不重要，倒也可以使用。反正哪一天不爽了用上面插件把图片批量搬回本地就好了。

也有插件或者软件实现图片自动上传图床，但是配置起来比较麻烦，而且现在靠谱的图床也不是很好找。喜欢折腾的同学自行研究吧。

工具的话推荐折腾 PicGo 或者 PicGo-Core，毕竟独立软件，在其他情况下也可以使用，而不只限于 Obsidian。

0023 今日待办和今日完成-多插件

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

每天的日报里显示两个栏目，一个今日待办，一个今日完成。

单纯查询待办事项和已完成事项都比较简单。举例（使用时请标记对应代码块的语言类型）：

任务相关的事情加一个 Tasks 插件会很方便，它也提供查询，而且语法很友好

Tasks 待办

Tasks 完成

但是这样并没有解决问题——今天。当然待办（未完成）这里可以宽泛一些，毕竟完成以后就到今日完成里去了。

那么今日完成呢？我们得把日期放入查询，就是必须这一天完成的才算。而且日期不能是动态的，得写死，不然每天看都是今天的日期……这符合我的使用习惯，但是每天新建一个日报就是另一回事儿了。日期应该写死为文档创建时的日期。

对了，用模板，在创建的时候生成日期写进去。所以用 Templater。（其实用 QuickAdd 也可以，不过大同小异，我只讲一个）。

下一个问题，程序怎么知道这个任务是哪天完成的啊？程序不知道，你得在完成任务的时候标记上是哪一天完成的。是不是很麻烦？没事儿，Tasks 插件中一个选项的事情，而且 Tasks 的标记方式和 Dataview 认可的标记方式完全相同，所以……很赞！
