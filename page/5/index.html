0008 插件和原则

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

我主要用 Dataview、Templater、QuickAdd 三个插件，这三个太强了，叹为观止。当然还会有一些实现小功能的插件，不一一列举，自己根据需求琢磨吧。

做一下简单的介绍，让大家有个初步印象：

  • Dataview 更强的搜索，并将搜索到的数据展示在笔记中。支持使用 JS 书写，但它也没说你必须搜索对吧，于是你可以利用它在笔记中运行 JS。

  • Templater 更强的模板，支持使用 JS 进行控制。模板是用来生成静态笔记的，也就是我可以用 JS 控制生成笔记（或者笔记片段）。而且可以插入动态模板（每次预览时重新渲染一次）。这自由度可以放飞想象。

  • QuickAdd 快速添加笔记，可以是往特定位置插入，也可以是新建笔记，还可以是运行一系列命令，甚至自己写脚本。

归结起来就是：都可以自己写代码，即程序员眼中的自由。

回来说原则，前面说过保持 Markdown 的纯洁性，这个原则很重要。

Dataview 是代码块形式，不污染其他内容，不过我只把它用在几个入口文件中。即时动态查询，非长久笔记，所以这种局部的插件依赖并不影响笔记整体的可移植性。

Templater 中的动态模板我也只用在入口文件上。而普通模板最终生成的是静态内容，只是方便，并不影响 Markdown 语法纯洁性。

QuickAdd 同上，只是更方便地去添加和创建笔记，并不污染笔记内容。

综上，只是提供便利，提升效率，而笔记内容并不对插件形成依赖。

可是，还是要讲，赞美这三个插件，笔记加上脚本和自动化，它，活了呀！

其他的，原则上尽量少用插件，避免奇怪问题，提升软件运行速度。如果需要插件，尽量优先核心插件，这样后续稳定性和兼容性更有保障。

以及反反复复讲的，笔记不依赖平台。

0007 利用工作区

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

这时候我有了两个入口文件：标签索引页和任务信息汇总页。打开笔记时这两个文件都可能被需要，那么左右分栏就可以了。因为这两个文件日常都不需要编辑，所以全都预览状态，两侧的侧边栏不需要，全都关掉。好了看起来很好，但是用一会儿就完全不是刚才的状态了，下次打开呢？随时想进入这种起始状态呢？

所以我们要把刚才设定好的这种状态保存一下，在需要的时候再调用出来。这个概念就是工作区（Workspace）。打开核心插件里的工作区，通过命令或者侧边栏按钮进行调用。

什么是命令？Ctrl+P 会出现一个输入框，可以输入并搜索各种指令，你搜索“工作区”，就会有对应的指令出来。

管理工作区，就可以输入名称保存当前状态了，以后还可以更新这个记录。当然，需要的时候就加载这个工作区就好了。

这样可以非常编辑的在几种工作状态下进行切换。利用的好效率会非常高，而且因为没有了繁琐的切换布局，思路不容易被打断，体验大幅度提升。

这种简单的小技巧，却又能带来大幅度体验提升的东西，特别喜欢！

0006 因何起始页

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

笔记还起始页，这好像和以往的认知不一样啊。

换个角度思考吧，笔记是干什么的？管理知识，管理信息，管理待办事项，管理碎片化内容……总之，是管理，没啥大问题。你不能说笔记就是用来记，光吃不吐那是貔貅啊，光记不用记它干啥，心理安慰？

所以，打开笔记软件，比较多的情况应该是阅读某些内容，查找某些内容，快速获得某些信息。当然，也有快速输入记录某些一些内容（这咕噜咱们后面聊）。

前面我说了自制标签索引页，那如果用这个页面做起始页，我就可以在打开笔记之后快速的定位到自己需要的某一类信息，然后找到自己需要的内容。

当然，这只是个举例，根据你的使用情况，可能还有各种其他用法。比如有人每天新建一个日记（日志、日报、log……），在里面汇总一天的信息。比如：今日待办，今日完成，本月目标等等，反正根据自己需求来吧。

我比较简单，不需要每天一篇笔记的日报，所以就只用一篇笔记去汇总信息。比如工作任务列表里标着 #Next 标签的任务，我用这个标签来标记打算下一步完成的任务。然后生活列表里有这个标签的任务，购物列表里有这个标签的内容，还有等着收的快递……为了美观加个图片，既然是入口，加个日历、天气、年进度，等挺合理的吧。大概是下面这个样子的：

汇总，入口，线索。如果有这些需求，那么可以考虑弄一个适合自己的起始页。也算是个性定制，客制化什么的了。其实也不算复杂，后面会一点点拆解。

0005 往事今何在

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

笔记多了搜索是一项必须的技能，不过搜索并不是只有关键词一种方法。

前面讲的标签就是一种很重要的方式，我从标签索引页点击一下，就可以进入对应的查询，查看所有相关内容。

当然，直接在文件结构里翻查也是，检索文件嘛。

如果你的笔记够多，那么最好学学更高级的搜索语法，其实也不难，把问题说清楚就行。这里是官方的帮助文档。认真读一遍，尝试几次就能写了，十分钟的学习来提高搜索效率还是很值得的。

如果没有那么多笔记，建议先去做笔记吧，没有积累，光学技巧，就只是花里胡哨的。

至于英文文档，基本高中英文水平就能读了，而且还可以借助各种翻译工具，理解下来应该没啥问题。这都不想做就也没必要继续了。虽然可以等别人嚼过的（翻译的），但是软件更新了可能功能有变化，这时候肯定是官方文档最及时。

如果上面没能劝退你，并且你也学习了一下搜索语法，大概理解了一些，那么我们可以继续了。

一个情况：我经常会进行某个查询，我能不能收藏这个查询，要不然一次次输入太麻烦了。

写一个代码块，把语言标记为 query，内容就写你的查询，然后预览，你会看到它把查询结果直接列在笔记中了。

这样可以在某个路径（path）下查找具有某个标签（tag）的内容。

所以，就是把你要的查询讲明白给软件听。

这里涉及到一些细节：

Markdown 的纯净性

最基础的 Markdown 语法是非常简单的，当然可以实现的功能也非常少。我的理解，它核心的思路是纯键盘快速输入，同时具有简单格式标记。这很适合用作笔记，但面对复杂场景就变得非常羸弱。

有人觉得实在难以满足，于是语法有了扩展，比如表格、待办，确实在一些场景下这些东西是很常用的。但是用纯文本去书写表格的体验我不敢恭维。不过这些语法基本上已经被大家认可并兼容。问题也不大。而且在纯文本状态下的可读性也不低（表格比较简单的情况）。

还有一种是用代码块来实现的增强，比如流程图的 Mermaid，数据图的 charts 等。它们和上面的查询一样，用的是 Markdown 里面标准的代码块，只是标记为特定的语言。在支持的情况下，这些代码会被渲染成特殊的样子，如果不支持，则以普通代码形式显示。以代码形式显示的时候可读性并不高，好在不影响其他内容。

我前面说过我的笔记尽量不平台依赖，可以随时更换软件。这就要求保持语法的纯净，不乱用不规范的私有的语法。Query 语法是 Obsidian 特有的，我只在一些入口笔记中使用，这样更换软件不会带来多大的影响。

当然这只是我自己的原则，在这里提醒各位有这样一种坑，而你可以根据自己的情况设定自己的原则。

更强的查询

Obsidian 有一个插件 Dataview，可以实现非常厉害的查询。用法和上面的 query 类似，也是代码块，不过语言标记为 dataview，然后使用它自己的 query 语法，详情可以在它的帮助文档中阅读。

确实，有点复杂，毕竟为了强大嘛。然而，这个插件支持直接使用 JavaScript 语法，这对于会 JS 的同学来说就几乎没有学习成本了，一下子如鱼得水。所以我日常使用这种方式，后面也会分享一些查询代码，并做讲解。

笔记的整理，首先是文件夹，但是这并不灵活，一方面深层文件夹不好操作，另一方面一个文件只能属于一个文件夹。

所以最好不要用深层文件夹，只是一层的文件夹也方便使用双链直接创建文件：[[Note/大老鼠怎么那么可爱捏]]。

然后添加元数据（meta），或者叫做 Front matter，反正就是笔记开始的一系列信息。有一些插件可以帮助，不过我推荐使用模板，Obsidian 自带的就可以。我的模板大概这样：

主要就是创建时间（created 或者 ctime）和更新时间（updated），更新时间这里不会自动更新，需要借助插件，比如：Update time on edit plugin。这些数据方便我们后期归类、查找笔记。

然后了解一个概念：别名（alias），虽然表面是是笔记的别名，其实你也可以理解为虚拟路径。一个文件可以有多个别名，如果运用得好，完全可以当作分类来使用。别名可以在元数据中进行设定（aliases）。

还有一个重要的概念是标签。标签非常灵活，可以是分类、关键词，甚至利用嵌套标签来表达层级关系（比如上面中带有斜线 / 的标签）。但下一个问题随之而来——标签多了不好管理。

也要有自己的组织。虽然软件可以列出所有的标签，但是平铺直叙，是没有任何线索的（字母或者数量排序）。我们可以自己建立一个标签页面，登记自己用到的所有标签，并加以归类，比如这样：

0003 同步什么的

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

Obsidian 的插件和主题都是在 Github 上，所以安装需要能够顺利的访问 Github。

Obsidian 官方提供同步和发布服务，但是收费，同步 8 美元/月，发布 16 美元/月，需按年购买。我买不起。（欢迎赞助： 老鼠爱发电）

不过前面说了，就是文件而已，那用网盘同步也完全没有问题的。以及其他各种文件同步工具和方式都没问题。

有一个第三方插件：Remotely Save，可以更方便的进行同步，支持 S3 云服务，Webdav 服务，OneDrive，Dropbox。这里我推荐搭配 TeraCLOUD 使用，这个网盘可以开启 Webdav 服务。我的邀请码是： 5RZJD，输入之后你我都可以获得几个 GB 的临时空间。免费空间可能是 100GB，记不清，反正一般做笔记足够用了。

任何云服务都不是绝对保险的，任何存储设备都不是绝对保险的。想要数据安全，就要在更多地方做更多的备份。

记得每次在开始使用前先同步一次数据，使用后再同步一次数据，避免数据冲突，这个习惯挺重要的。

手机上如何同步？

上面的方法基本适用。

但是，同步这么麻烦，而手机上并不适合输入和编辑，这时候要认真的去思考一个问题——真的有必要和手机同步么？

我承认，有些复杂的需求下，确实需要，那么去折腾吧，以及牢记各种操作规范，保障数据安全。

而如果你只是想方便在手机上阅读，也许可以考虑一下发布服务。有好几款插件可以实现，十分简单，右键——发布，就发布了，都不需要注册。我目前用的：Obsius Publish，发布之后还能进行简单的管理。类似的还有： OzanShare Publish，私密性可能更高一点。当然也可以自己搭建。

如果你是想记录闪念，那手机的记事本也不错了；收集网址，浏览器的收藏夹也不错。

笔记是需要整理的，如果收集之后必然要有整理过程，那么收集在什么地方倒不那么重要了。

如果是用来管理待办、任务，也许可以考虑新建一个保险库（Vault），手机上只同步这个保险库，这样万一出现问题，也不会影响到其他数据。而且也不至于每次都需要同步并不需要的数据。但，为什么不用个专业的 GTD 软件呢？

0002 笔记是什么

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

怎么比喻都行，反正大家用法各不相同，没必要非得分个高下，也没必要非找个所谓最好的。适合自己的才是最好的。

随便分几种情况聊聊，并不全面，仅供参考：

日志类

日记呀，或者工作记录呀，或者流水帐啊，或者闪念啊。标记好日期有个索引的线索就好，可以日后翻看，能对应上日期就够了，毕竟再复杂的线索恐怕也难以归类出来。必要的话加个相关标签。

知识类

知识需要有内化的过程，体现在笔记上就是不断地翻看整理。这里觉得 Obsidian 的漫游功能（核心插件，需要开启）挺适合用来回顾笔记的。

然后，笔记不整理和没记没区别。那种看见啥都复制、粘贴、艾特xx笔记的做法就……咱说实在的，直接下载个维基百科离线版可能更好。

整理啊，回顾啊，这比记录还要重要很多的。同时仔细地分类笔记，添加标签等，这些都是后期索引的线索。

网络摘抄

先收入笔记，然后抓时间整理，最好用自己的语言再复述一遍。然后归入自己的知识库之中。

收集箱

闪念、摘抄这些都是快速记录的，后续应该有整理过程的，所以这些内容先进收集箱临时存储，等待后续整理，然后归档到知识库之中。

如此，一套笔记法就出来了。有收集、整理、回顾，也有分类、组织、管理。

简单吧，但长期践行下去就是考验耐心和毅力的事情了。

安装，打开，写笔记，就行了。

还要怎样？这就是个工具，工具就该有工具的样子，用呗，用熟了就好用了。工具折腾出花儿来，回头一看就写了两篇笔记，也没啥意思。

Obsidain 手机端几乎完整照搬电脑端，真没啥学习成本了。

电脑端在保险库（vault）打开界面中可以设置软件语言，最开始我不知道，稀里糊涂用了很久英文版。手机端在设置的关于里面可以设置界面语言。这语言设置多少有点不按套路出牌。

设置里的每个功能都有说明，看不懂的就不动就行了。

保险库就是一个文件夹，然后你的笔记都保存在里面，然后用网盘备份这个文件夹就可以同步和备份了。当然可以有多个保险库，看具体需求了。

既然是文件，那么就可以按文件结构去管理笔记，就是一个文件夹放一类笔记。如果你的笔记比较多，这里值得规划一下。如果刚开始，那么先写，先有了记笔记的习惯再说其他的。

Markdown 语法有十几分钟就学会了，就学最基础的就行。

如果你想让一件事情变成长久的习惯，就一定不要把它搞复杂。

0000 原本无一物

  • 《遇见 Obsidian》系列笔记汇总目录
  • 如果觉得有用就支持一下作者：老鼠爱发电

我不喜欢笔记，我以前从来不记笔记，我的记忆就是我的笔记。

但是，记忆这东西……有一天会叛逃，靠！釜底抽薪了这是

不想讲知识管理啥的，没那么高大上，我管理的有些不是知识，我只是不想忘记钥匙放哪里了。

为啥是 Obsidian？

都行，我真没所谓，记事本也行，只不过各种意义上的不好用罢了。VS Code 也行，Typora 也行……我还能随口讲几十个（摸摸脑袋，好像又没记住那么多名字）。

不过这里涉及到了我笔记的一个重要原则：绝不依赖特定平台。免得我把我的知识啊，人生啊，一切的一切写在上面的，以为自己忘不掉了，然后它收费了，或者倒闭了。我以为笔记是我的第二大脑，哪承想第二大脑也叛逃了。

所以我的笔记是 Markdown 格式，本质是文本文件，用标记添加简单的格式，用记事本也能读，而且可读性还不错。想怎么存储怎么存储，想怎么同步怎么同步。

至于 Obsidian，就是个管理/编辑工具而已，随时可以更换，甚至我现在日常也用它和其他编辑器配合。

行了，结帖。

Obsidian 香啊！

无论是 Dataview 插件还是 Templater 插件，都可以自己写 JS，恰好，我会写 JS。巧么了，这不是。

都能自己写脚本了，而且还是我会的语言，那就约等于没啥学习成本，还拥有最大的自由度。我当然觉得香了。

这个系列，只是从我的角度记录我的使用，分享其中的舒爽和乐趣。还有一些插件用法和我写的小脚本。
